## 三、已留存資產（呂同學交接）

呂同學離隊前已將其個人專題核心經驗交接給姜，並將納入產品設計:
- **用戶自帶 LLM API key** 機制
- **每次任務的算力消耗「預估值 + 實際值」顯示**
- **依任務推薦適合的 LLM 模型與當前計價方式**

> 策略價值:此「成本透明」設計恰好打中本產品 TA（尤其接案/顧問型 PM）對「費錢、不確定划不划算」的痛點，是潛在差異化亮點。建議納入 MVP 或至少「即將開放」展示。

## 四、關鍵風險與策略洞察

1. **三受眾、三敘事**:本專案有三個時間點面對三種受眾——平時對蜜蜂老師（把關方法論）、7/30 對投資人（投資價值）、7/30 對雇主（成員能力）。MVP 與報告須「一份成果、三種敘事」:給評審看方法論扎實、給投資人看市場與擴張、給雇主看團隊執行力。
2. **競爭同儕的雙重利用**:6 組同儕既是問卷樣本又是競爭對手，訪談時保持互助以確保配合度，報告日則以差異化脫穎而出。
3. **分工尚未定案**:目前僅蔡同學的組長角色明確，其餘成員正式分工待 WBS 後確立，避免過早綁定。
